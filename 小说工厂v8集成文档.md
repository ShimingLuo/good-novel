### 2. 工作流层级

小说工厂采用 **L0-L5 六层流水线**架构：

```
L0  项目初始化   →  三层问答 + 命名 + 创作宪法
L1  创意引擎     →  并行创意策划书
L1.5 竞品研究   →  搜索分析同题材热书（必过Gate）
L2  世界工厂     →  人物/世界/势力设定
L3  大纲农场     →  全书结构 + 双轨细纲
L4  正文流水线   →  状态机 + 独立审核
L5  番茄发布     →  发布包生成
```

### 3. Gate门禁系统

| Gate | 触发时机 | 检查条件 |
|:----:|:---------|:---------|
| **B** | 进入L1.5前 | 创意审核≥6分 |
| **S** | 进入L3前 | 设定审核≥6分 |
| **A** | 进入L4前 | 六维品控全部通过 |
| **C** | 写第N章前 | 前章status=passed |
| **E** | 每层完成后 | 交付物清单完整性 |

## 📊 核心技能详解

### 1. 审核引擎（11维审核）

审核引擎采用 **评分锚点参照系**，确保评分的客观性和可比性：

| 分数 | 含义 | 读者体验 |
|:----:|:-----|:---------|
| 9分 | 惊艳 | 读者会截图发朋友圈 |
| 8分 | 亮点 | 读者愿意评论追更 |
| 7分 | 基准线 | 读者不会弃文 |
| 6分 | 可改进 | 读者感觉不对劲 |
| 5分 | 有问题 | 读者明显出戏 |
| ≤4分 | 严重问题 | 读者大概率弃文 |

**审核维度**：
- ① 错别字与语法
- ② 逻辑一致性
- ③ 人设一致性（8个陷阱问题）
- ④ 伏笔照应
- ⑤ 设定一致性
- ⑥ 文风一致性
- ⑦ 断章质量
- ⑧ 情感曲线
- ⑨ 违规检查（一票否决）
- ⑩ AI味检测
- ⑪ 追读力检查

### 2. 反AI规则系统

基于朱雀AI检测器的实测数据，包含 **18条反AI规则**：

**核心规则**：
- R09: 写后不解释
- R10: 内心活动转为行动
- R11: 短句堆叠限制（≤3句）
- R12: 对话后禁止加心理注解
- R13: 禁止"环视+评估"模板
- R14: 禁止"眼神/面部解读"模板
- R15: 禁止三重否定句链
- R17: 紧张场景禁用比喻
- R18: 禁止"内心运转/猜测链"模式

**黑名单词汇**：
- 神态微动作：嘴角抽了抽、眼角跳了跳
- 形容词黑名单：深邃、凌厉、淡然、复杂
- AI比喻模板：像...一样、仿佛...
- 陈词滥调：不知为何、莫名的、下意识的

## 🏗️ 架构特点

### 1. Hybrid混合架构

小说工厂采用 **主Agent + 子Agent** 的混合架构：

**主Agent职责**：
- ✅ 与用户对话
- ✅ 轻量内联操作（字数预算、状态更新、快照创建）
- ❌ 禁止自行审核、跳过审核

**子Agent职责**：
- 写作Agent：只写正文，写完即停
- 审核Agent：只审章节，零写作上下文
- 修复Agent：独立修复问题

### 2. 状态机管理

通过 `pipeline_state.yaml` 全局状态文件管理流程：

```yaml
project: "作品名称"
layers:
  L0: completed
  L1: completed
  L2: completed
  L3: completed
  L4: in_progress
  L5: pending
workflow:
  current_layer: "L4"
  current_chapter: 5
  last_action: "write_ch5"
  last_status: "awaiting_review"
```

### 3. 9维真相档案

小说追踪系统维护9个维度的数据：

1. 角色关系.yaml - 角色卡（YAML结构化）
2. 时间线.yaml - 章节时间记录
3. 伏笔清单.yaml - 伏笔埋设与回收追踪
4. 资源账本.yaml - 物品/金钱/物资
5. 情感弧线.yaml - 角色情感变化轨迹
6. 支线进度.yaml - A/B/C线状态
7. 世界状态.yaml - 当前世界状态基线
8. 因果链.yaml - 因果约束追踪
9. 信息边界.yaml - 信息不对称追踪

## 🚀 使用方式

### 1. 切换到小说工厂套件

在 good-project 前端界面中：
1. 打开技能套件管理器
2. 选择 "小说工厂 v8.0"
3. 激活套件

### 2. 启动新项目

使用"一键小说工厂"技能：
```
用户输入：写一本悬疑无限流小说，主角是重生者，核心冲突是解密
```

系统将自动执行：
1. L0 项目初始化（三层问答）
2. L1 创意引擎（并行策划）
3. L1.5 竞品研究（搜索分析）
4. L2 世界工厂（设定生成）
5. L3 大纲农场（细纲设计）
6. L4 正文流水线（逐章写作）
7. L5 番茄发布（发布包生成）

### 3. 选择交互模式

| 模式 | 适用人群 | 特点 |
|:-----|:---------|:-----|
| 极速模式 | 新手 | 零参与，一次性交付 |
| 监控模式 | 老手（默认） | 每章推送，异常通知 |
| 精细模式 | 专业作家 | 每章确认，严格门禁 |
